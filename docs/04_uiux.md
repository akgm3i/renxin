# RENXIN UI/UX

## 1. 主要画面
- ホーム
- プレイヤー管理
- ゲーム（ボード入力/スコア/UNDO）
- 練習モード（練習メニュー実行）
- 練習メニュー
- ラウンドリプレイ
- 観戦ビュー
- 統計
- 設定

## 2. 画面遷移ルール
- 起点はホーム。
- ゲーム終了（ラウンド）でリプレイへ遷移。
- 練習モード（練習メニュー実行）では明示終了まで継続し、任意タイミングでリプレイ確認可能。
- 練習メニューはセッション単位で開始/終了できる。
- リプレイからゲーム/ホームへ戻れること。
- 設定はホームから常時到達可能。
- 01/Cricket結果画面から練習メニューへ遷移可能。

## 3. 入力原則
- タッチ優先の大きな操作領域。
- 入力結果（スコア）を即時表示。


## 4. 主要ユーザーフロー（初期）
### 4.1 01/Cricket→練習メニュー遷移
1. ゲーム終了
2. 弱点サマリ表示（例: Double率、Checkout率）
3. 推奨メニュー表示（理由付き）
4. ユーザーが開始または別メニューを選択

### 4.2 練習メニュー実行
1. メニュー選択
2. セッション開始
3. 投擲入力 + 途中リプレイ + UNDO
4. 終了時に達成度/推移を保存・表示

## 5. UX受け入れ条件
- 推奨メニューの理由が1行で説明される。
- 練習セッション終了時に、前回比（改善/悪化）が確認できる。
- 誤タップ時に1操作でUNDOできる。
