# RENXIN 運用仕様

## 1. 設定既定値
- video_resolution: 1280x720
- video_framerate: 30
- auto_delete_days: 90
- video_storage_limit_gb: 80
- rpc_port: 8080

## 2. 代表的な障害時挙動
- カメラ未認識（起動時）: 起動ブロック + 設定誘導。
- 容量不足（録画中）: 録画停止 + 通知。
- DB破損: 復旧ダイアログを表示。
- ポート競合: ポート変更を案内。

## 3. ログ
- tauri-plugin-log を利用。
- ERROR/WARN/INFO/DEBUG を出力。
- 診断情報エクスポートを提供。


## 4. バックアップ/保守
- DBは日次でローカルバックアップを作成（世代: 7）。
- 動画インデックスと実ファイルの整合性チェックを週次実行。
- 診断エクスポートには `logs + config + latest_stats_snapshot` を含める。


## 5. リリース前チェック
- 2カメラ30分連続録画テストを実施済み。
- 容量上限到達時の劣化動作（録画停止/通知）を確認済み。
