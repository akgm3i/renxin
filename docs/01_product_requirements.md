# RENXIN プロダクト要件

## 1. 目的
RENXINは、ダーツ練習の記録・振り返り・共有を支援するデスクトップアプリである。

## 2. 想定ユーザー
- 主なターゲットはプロジェクトオーナー本人（単独利用）とする。
- 将来的な複数ユーザー利用は拡張要件として扱う。

## 3. 確定事項
- ハードウェア: タッチ対応PC + 2カメラ（姿勢/ボード）。
- 実行基盤: Tauri（React + TypeScript + Rust）。
- 運用方針: ローカルファースト。

## 4. 機能要件
### 4.1 ゲーム進行
- 01 / Cricket の入力・計算・保存。
- 複数プレイヤー管理。
- 1投単位の履歴とUNDO。

### 4.2 リプレイ
- 1ラウンド（3投）終了後の即時再生。
- 通常再生 / 0.5x / 一時停止。

### 4.3 観戦
- LAN内ブラウザでスコアのリアルタイム閲覧。
- 動画配信は段階導入可（先にスコア）。

### 4.4 統計・レーティング
- PPD / MPR 算出。
- DARTSLIVE公式準拠。
- `dual`（80%公式値 + 100%参考値）を許容し、主表示は公式値。

### 4.5 練習モード（＝練習メニュー実行モード）
- 練習モードは、選択した練習メニュー（Around the Clock / Bob’s 27 / 121 等）を実行するためのモードとする。
- セッション単位で開始/終了し、投擲履歴・スタッツ・メニュー別達成度を保存できる。
- 練習モード中もラウンドリプレイとUNDOを利用できる。
- 01/Cricket本番モードとはスコア保存単位を分離し、比較分析できること。


### 4.6 練習メニュー提供（01/Cricket強化向け）
- 01/Cricket本番モードとは別に、基礎能力を鍛える練習メニューを提供する。
- 初期メニュー候補: Around the Clock（精度）, Bob’s 27（ダブル精度）, 121 Checkout（上がり判断と決定力）。
- 01/Cricketの結果画面から、弱点（例: ダブル率低下、Checkout失敗）に応じた練習メニューへ遷移できること。
- 練習メニューは1セッション単位で結果保存し、推移を確認できること。

## 5. 非機能要件
- タッチ操作が遅延なく行えること。
- ラウンド終了から5秒以内に再生画面へ遷移できること。
- 2カメラ同時録画時も安定動作すること。
- 将来のサーバー分離に耐える構造であること。


## 6. 要件レビュー時の確認項目
- 入力精度要件（タッチ誤差許容、補正方法）は明確か。
- レーティング更新タイミング（都度/ゲーム終了時）は明確か。
- 練習モードの終了条件と保存単位は明確か。
- オフライン時の挙動（保存失敗時リカバリ）は明確か。
- 観戦機能の最小範囲（スコアのみ/動画含む）は段階定義されているか。
- 練習メニューのレコメンド基準（どの指標で何を勧めるか）は明確か。


## 7. Decision Needed（将来チューニング項目）
- 終了条件: 手動終了のみ / 投数上限 / 時間上限 / ラウンド上限。
- 保存粒度: 1投ごと即時保存 / ラウンド単位保存 / 終了時一括保存。
- リプレイ導線: 毎ラウンド自動遷移 / 任意ボタン遷移。
- 練習プリセット: 01 / Cricket / Bull / セグメント練習の初期提供要否。
- 録画失敗時の扱い: スコア練習継続可否、通知レベル。
- 成果指標: PPD/MPRに加えて表示したい指標（命中率、分布など）。
- 練習メニュー候補の最終採用数と卒業基準。


## 8. 技術要件・構成・設計の妥当性評価（現時点）
- 妥当: ローカルファースト + Tauri構成は、単独利用・オフライン重視の要件に一致する。
- 妥当: UI/Core分離と将来サーバー分離前提は、拡張余地を確保できる。
- 要具体化: 非機能要件（遅延なく、安定動作）を計測可能な閾値へ分解する必要がある。
- 要具体化: レーティング計算の丸め規則・更新タイミング・表示優先順位をデータ契約へ固定する必要がある。
- 要具体化: 練習メニュー推奨ロジック（弱点指標→推奨メニュー）を規則化する必要がある。


## 9. 非機能要件の測定基準（受け入れ条件）
- 入力遅延: タッチ入力からUI反映まで 100ms 以下（p95）。
- リプレイ遷移: ラウンド終了から再生開始まで 5秒以内。
- 安定性: 2カメラ同時録画30分でアプリクラッシュ0件。
- リソース上限: 2カメラ録画時の平均CPU使用率 80%以下（計測条件を別紙で固定）。
- 観戦遅延: LAN観戦クライアントへのスコア反映遅延 500ms 以下（p95）。


## 10. 練習モード既定ポリシー（実装開始時の初期値）
- 終了条件: 手動終了を既定とし、投数/時間上限は将来拡張。
- 保存粒度: 1投ごと即時保存。
- リプレイ導線: 任意ボタン遷移（自動遷移しない）。
- 初期プリセット: Around the Clock / Bob's 27 / 121 Checkout の3種。
- 録画失敗時: スコア記録は継続し、WARN通知を表示。
- 成果指標: 命中率、平均得点、前回比を表示。
